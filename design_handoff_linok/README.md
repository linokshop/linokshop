# Handoff: ЛінОк — інтернет-магазин рибальських товарів

## Overview
Багатосторінковий сайт рибальського магазину «ЛінОк» (слоган «Там, де клює — там я!»).
Ключова особливість — державна програма **«Ветеранський спорт»** (оплата спорядження
ветеранами через застосунок «Дія»), яка присутня майже на кожній сторінці: верхня
плашка-анонс, пункт меню, банер у hero, окрема сторінка й спосіб оплати в кошику.

Мова інтерфейсу — **українська** (у шапці є перемикач UA/RU, наразі візуальний).

## About the Design Files
Файли в цьому пакеті — це **дизайн-референси, зроблені в HTML** (прототипи, що
показують вигляд і поведінку), **а не продакшн-код для копіювання напряму**.
Завдання — **відтворити ці макети у вашому цільовому середовищі** (React / Next / Vue /
тощо) за прийнятими у проєкті патернами й бібліотеками. Якщо середовища ще немає —
оберіть доречний фреймворк (рекомендація: **Next.js + React**, оскільки це e-commerce
з маршрутизацією між сторінками) і реалізуйте дизайн там.

Формат вихідних файлів — «Design Components» (`.dc.html`): кожен файл має власний
`<template>` з інлайновими стилями та (де є) клас логіки `Component`. Це зручно читати
як специфікацію розмітки, але переносити варто у звичайні компоненти вашого стеку.

## Fidelity
**High-fidelity (hifi).** Кольори, типографіка, відступи й компонування — фінальні.
Відтворюйте UI піксель-у-піксель засобами свого дизайн-системи/бібліотек.
Зображення товарів — плейсхолдери (діагональна штрихована заливка з підписом «фото»),
їх замінить замовник реальними фото.

## Interactions & Behavior
- **Навігація**: усі переходи — звичайні `<a href="...">` між сторінками. У реальному
  застосунку замініть на маршрути (Next `<Link>` / React Router).
- **Бургер-меню** (Header): тап по кнопці тогглить стан `open` → показ/приховання мобільного списку.
- **Клікабельні**: картки категорій/товарів → product; банери програми → veteran; промо → promos.
- **Наразі не реалізовано (треба зробити в проді)**: реальний кошик (додавання/видалення/лічильники),
  фільтри й сортування каталогу, пошук, форми (валідація й сабміт), перемикач мови UA/RU,
  реальна оплата й інтеграція з «Дія», акордеон FAQ.
- **Респонсив**: реалізовано через `@media` у `<style>` кожного файлу з `!important`
  (перекриває інлайнові стилі). Брейкпоінти: **1024px** (планшет), **900px** (складання
  двоколонок / бургер), **600–640px** (телефон), точково **420/700px**.

## State Management
Потрібні стани для продакшн-реалізації:
- **Кошик**: список позицій `{id, name, category, option, price, qty}`, похідні суми
  (проміжна, знижка, доставка, до сплати), дії add/remove/updateQty/applyPromo.
- **Каталог**: обрані фільтри (категорії[], ціна {min,max}, бренди[], inStock), сортування,
  сторінка пагінації; похідний відфільтрований/відсортований список.
- **Картка товару**: обрана опція (тест), кількість, обране фото галереї, wishlist-стан.
- **Header**: `open` (мобільне меню), `lang` (UA/RU).
- **Форми** (контакти, отримувач у кошику): значення полів + валідація.
- **Дані**: товари, категорії, акції, новини — зараз захардкоджені (див. `catalog` logic-клас
  і плейсхолдери), у проді тягнути з API/CMS.

### Типографіка
- **Oswald** (400/500/600/700) — заголовки, UI-акценти, uppercase, letter-spacing ~.02–.06em.
- **Bitter** (600/700/800, serif) — заголовки на **світлих** сторінках (about/delivery/contacts).
- **Golos Text** (400–800) — основний текст/абзаци.
- Джерело — Google Fonts (див. `<link>` у кожному файлі).
- Приблизна шкала: H1 hero 62–78px, H1 сторінок 50–54px, H2 секцій 32–40px, body 15–18px,
  дрібний/мета 12–14px. На телефоні заголовки зменшено (див. media-правила).

### Spacing / радіуси / тіні
- Контейнер: `max-width:1320px; margin:0 auto`, паддінги секцій ~40–72px (десктоп),
  на мобільному зменшено до 24–28px.
- Радіуси: картки 6–14px, кнопки 4–8px, круглі елементи 50%.
- Тіні: мінімальні; у hero-картці програми `0 26px 70px rgba(0,0,0,.45)` та text-shadow для
  читабельності тексту поверх фото.
- Сітки — `display:grid; gap:14–24px`; ряди — `display:flex; gap:...` (не покладатись на margin).

розмітку в компоненти, інлайнові стилі — у вашу систему стилів (CSS-модулі / Tailwind / styled),
а `@media`-блоки з `<style>` — у відповідні брейкпоінти. Рантайм `support.js` — службовий,
у прод не переносити.
